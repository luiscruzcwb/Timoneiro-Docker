Tudo que você precisa

Projetado para ser simples de instalar e poderoso em produção.

Interface web completa

Dashboard com status em tempo real via WebSocket, histórico de atualizações, aprovação manual e rollback com um clique. Sem linha de comando necessária para o dia a dia.

Leve

Binário único em Go, imagem Docker abaixo de 30 MB. Roda confortavelmente em um Raspberry Pi.

Multi-ambiente

Monitore hosts Docker remotos via TCP, socket Unix ou agente Timoneiro leve. Toda a infraestrutura em um painel.

Registries privados

Docker Hub, GHCR e qualquer registry OCI: Harbor, Nexus, Gitea. Credenciais armazenadas localmente.

Notificações

Discord, Telegram, Slack, SMTP, Gotify, ntfy e qualquer serviço Shoutrrr. Múltiplos canais simultâneos.

CVE scan

Integração com Trivy para detectar vulnerabilidades em novas versões antes de aplicar atualizações.

Políticas de atualização

Modos Manual, Automático e Agendado. Janelas de manutenção por dia e horário. Exceções por container ou por stack Compose. Nenhuma atualização surpresa em produção.

API REST + WebSocket

API completa para automação e integração com pipelines CI/CD. WebSocket para eventos em tempo real. O mesmo endpoint que a interface web usa.

# Listar containers com atualizações disponíveis
GET /api/containers?status=update_available

# Receber eventos em tempo real
WS ws://localhost:8080/ws
"type": "update.available", "container": "postgres"

Em funcionamento em minutos

Sem configuração complexa. Aponte para o socket Docker e pronto.

1

Instale o Timoneiro

Execute com Docker Run ou Compose. Monte o socket do Docker e um volume para persistência.

docker run -d \ --name timoneiro \ -p 8080:8080 \ -v /var/run/docker.sock:/var/run/docker.sock:ro \ -v timoneiro_data:/data \ ghcr.io/luiscruzcwb/timoneiro
2

Acesse a interface

Abra http://localhost:8080 no navegador. O Timoneiro descobre automaticamente todos os containers em execução.

3

Configure as políticas

Escolha o modo de atualização (Manual, Automático ou Agendado), defina janelas de manutenção e configure os canais de notificação.

4

Relaxe

O Timoneiro monitora seus registries e mantém seus containers atualizados, ou te avisa quando há algo novo para revisar.
